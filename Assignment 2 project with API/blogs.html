var-> Variables declared with var are function-scoped or Globally scoped. They are visible throughout the entire function, regardless of where they are declared. Can be re-declared within its scope. Can be updated within its scope. Hoisted to the top of its scope.
let->Variables declared with let are block-scoped. They are visible only within the block where they are defined. Cannot be re-declared within the same scope. Can be updated within its scope. Hoisted to the top of its scope, but not initialized until the declaration is reached.
Const-> variables declared with const are block-scoped. Cannot be re-declared within the same scope. Cannot be updated once initialized. Hoisted to the top of its scope, but not initialized until the declaration is reached.

Use Cases for null: Null is used to indicate the absence of a value. Setting a variable to null is sometimes used to reset or clear a value.
Use Cases for undefined: Represents the absence of an assigned value. When trying to access an object property that doesn't exist, the result is undefined. Variables that are declared but not assigned a value are automatically initialized with undefined. If a function doesn't have an return statement, it returns undefined.

Rest API-> REST API, which stands for Representational State Transfer Application Programming Interface is a way for computer programs to talk to each other over the internet. It follows principles like using standard web addresses (URLs) for things, using standard commands like GET for getting data, and not keeping track of past interactions. REST makes it easy for different programs to work together in a simple and scalable way.
